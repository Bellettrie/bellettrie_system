How to use Search

The search function tries to match the search query to words in the title of books, series, as well as the names of authors. It also matches on parts of books.
It will display all books that match to all words in the query.
There are two special cases to keep in mind:
Book codes: The system will search for book codes, but only if it's fed only one word. So do not put spaces in the query if you want to find something.
It will try to complete though, so, for instance, SF-T-37 will find all Tolkien books, and possibly some more.
Regex-like operators: The system supports some helpers to allow the finishing of incomplete words. These are: asterisk (*), plus (+), question mark (?).
These characters state that the word should be longer, but that the user doesn't know what should be there. Underneath is a simple table of what's happening:
OperatorMatches on count
?Exactly one character
*Zero or more characters
+One or more characters

Please take care not to write too long queries. However, if you don't know how to spell "Hitchhiker's Guide to the Galaxy", a simple "Hitch*" will suffice.
Note that other regex commands are not supported. {% endblock %}
